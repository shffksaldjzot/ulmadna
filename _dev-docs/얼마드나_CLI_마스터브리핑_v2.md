## 2. 핵심 견적 산출 방식 — 하이브리드 적산

### 2-1. 두 가지 타입이 공존

**A타입 (개수 적산)** — 개수로 떨어지는 공정
```
욕실 = 양변기(단가×1) + 세면기(단가×1) + 수전(단가×1) + ... = 소계
      소계 × 개소수 = 공정 합계
```
해당 공정: 욕실, 주방(싱크대), 에어컨, 중문, 방문, 조명기구, 신발장, 붙박이장, 아트월

**B타입 (면적 품셈)** — 일반인이 면적을 모르므로 평수로 자동 계산
```
도배 = 평당 단가 × 평수 = 공정 합계
바닥 = 평당 단가 × 평수 = 공정 합계
```
해당 공정: 도배, 바닥, 타일(현관/주방/베란다), 철거, 몰딩/걸레받이, 필름(일부), 도장

**혼합** — 한 공정 안에 A+B 혼합
```
욕실 = 도기(A타입: 양변기 1개, 세면기 1개) + 타일(B타입: 면적 기반) + 인건비(일당×일수)
```

### 2-2. 총액 공식

```
총액 = Σ(각 공정 금액) × 지역계수(1.0 고정) × 시공환경계수 + 예비비(10%)
```

### 2-3. 인건비 계산 규칙

```
인건비 = 일당 × 인원 × 소요일수
기본: 2인 기준
예외: 소요일 0.5일 이하 → 1인으로 전환
```

인건비는 JSON의 `labor_rates`에서 직종별 일당 참조.
현재는 B타입 공정의 시공비가 평당 단가에 이미 포함되어 있으므로, 별도 인건비 계산은 Level 3(아이템 전부 열림)에서만 표시.

### 2-4. 기업이윤

JSON의 `config.margin_rate_range`에서 min 5% ~ max 10%.
기본값 5% 적용. 사용자가 슬라이더로 조절 가능.

## 4. UI 요구사항

### 4-1. 원페이지 구조

**스텝퍼 UI 금지.** "다음" 버튼 누르는 방식 절대 안 됨.
한 페이지에서 스크롤하며 모든 조건 선택 + 결과 실시간 갱신.

### 4-2. 레이아웃

**데스크톱:** 좌측에 입력(공정 선택, 옵션), 우측에 결과(총액, 공정별 비중 차트) 고정
**모바일:** 상단 입력, 하단 결과. 결과 영역은 하단 고정 바(floating bar)로 총액 항상 표시. 탭하면 상세 펼침

### 4-3. 실시간 반응

모든 선택 변경 → 총액 즉시 재계산. 버튼 클릭 없이 자동 반영. debounce 200ms 적용.

### 4-4. 결과 출력

- 공정별 금액 리스트
- 공정별 비중 차트 (도넛 또는 수평 바)
- 숨은 비용 경고 (JSON `hidden_costs` 참조)
- 절약 팁 (JSON `tips` 참조)
- PDF 다운로드 (개인정보 입력 없이 바로)
- 카카오톡 공유 (결과 카드 이미지)

## 7. JSON 데이터 사용법

### 7-1. 파일 위치

```
/src/data/ulmadna_db.json
```

### 7-2. 데이터 구조 핵심

```typescript
interface Process {
  id: string;           // "bathroom", "wallpaper", ...
  name: string;         // "욕실", "도배", ...
  type: "A_item" | "B_area";  // 적산 방식
  options?: Option[];   // 등급별 옵션
  items?: Item[];       // Level 3 아이템 상세
  presets?: Record<string, Preset>;  // Level 1 등급별 프리셋
}

// A타입: 개수 적산
bathroom_total = items.reduce((sum, item) => sum + item.price * item.qty, 0) * num_units;

// B타입: 면적 품셈
wallpaper_total = option.price_per_pyeong * pyeong;
```

### 7-3. 등급 선택 → 프리셋 매칭

```typescript
const grade = "mid"; // 사용자가 "중급" 선택

processes.forEach(process => {
  if (process.presets?.[grade]) {
    // 프리셋 금액 사용
    total += process.presets[grade].total;
  } else if (process.options) {
    // 옵션에서 grade 매칭
    const matched = process.options.find(o => o.grade === grade);
    if (matched && matched.price_per_pyeong) {
      total += matched.price_per_pyeong * pyeong;  // B타입
    } else if (matched) {
      total += matched.price;  // A타입
    }
  }
});

// 이윤 + 예비비
total = total * (1 + margin_rate) * (1 + contingency_rate);
```
