## 0. 최상위 대원칙 (먼저 읽을 것)

이 작업은 **세 가지로 요약**된다:

1. **A. 광고 슬롯 4개를 새로 추가한다.** (이 문서 §3~§6)
2. **B. v1의 기존 기능은 100% 그대로 보존한다.** (이 문서 §2)
3. **디자인(표현 계층)만 바꾼다. 로직(계산·상태)은 절대 건드리지 않는다.** (이 문서 §1)

CLI는 작업 중 길을 잃으면 이 3가지로 돌아온다. 의심스러우면 "기능 보존"과 "로직 불가침"이 항상 우선한다.

## 1. 디자인만 변경한다 — 표현 계층 / 로직 계층 분리 원칙

화면은 두 층으로 구성된다. 이 작업에서 **건드려도 되는 것과 절대 안 되는 것**을 명확히 가른다.

### 건드려도 되는 것 (표현 계층 / presentation)
- HTML 구조, CSS, 색상, 간격, 폰트, 둥근 모서리
- 컴포넌트의 화면상 배치·크기·정렬
- 새 광고 슬롯 컴포넌트 추가

### 절대 건드리면 안 되는 것 (로직 계층 / logic & state)
- 견적 계산 함수 및 그 입출력
- 상태 관리(state) — 평형/연식/등급/이윤율/공정 ON·OFF/자재 선택값/수량 등 모든 상태
- 이벤트 핸들러가 호출하는 함수, 그 함수가 읽고 쓰는 데이터
- API 호출, 데이터 흐름

### 작업 방식
- 새 UI 컴포넌트는 v1의 **기존 상태·함수에 *연결***되어야 한다. 기존 로직을 *재작성*하면 안 된다.
- 버튼이 화면 어디에 있고 무슨 색인지는 바꿔도 된다. 그러나 그 버튼이 눌렀을 때 호출하는 함수와 상태는 그대로 둔다.
- 권장 순서: (1) v1에서 계산·상태 로직을 건드리지 않을 영역으로 식별·격리 → (2) 그 위에 새 디자인 UI를 입힘 → (3) §2 인벤토리로 회귀 검증.

## 2. v1 기능 보존 인벤토리 (Definition of Done 체크리스트)

아래 모든 항목은 v2에서 **v1과 100% 동일하게 작동**해야 한다. 디자인 개편 완료 후, CLI는 이 목록을 한 줄씩 직접 클릭·조작하여 검증한다. 전 항목 통과해야 "완료"다.

### 2.1 입력부 — Step 01 기본 공간 정보
- [ ] 평형 선택: 18평(59㎡)/24평(79㎡)/25평·국평(84㎡)/30평(99㎡)/34평(114㎡)/40평(132㎡)/45평(151㎡) + 직접입력 — 8개 옵션 단일선택
- [ ] 연식 선택: 10년 미만 / 10~20년 / 20년 이상 — 3개 단일선택
- [ ] 등급 선택: 기본(가성비) / 중급(대중적) / 고급(프리미엄) — 3개 단일선택. **선택 시 전 공정의 기본 자재가 자동 설정되는 동작 보존**
- [ ] 기업 이윤율: 0% / 3% / 5% / 7% / 10% — 5개 단일선택
- [ ] 초기화 버튼 — Step 01 입력 리셋

### 2.2 입력부 — Step 02 공정 세부 선택
- [ ] 13개 공정 카테고리 전부 존재: 철거 / 확장·설비 / 전기·조명 / 에어컨 / 목공 / 창호 / 도배·바닥 / 도장·필름 / 타일 / 욕실 / 주방 / 가구 / 부대비용
- [ ] 각 공정 카드: ON/OFF 토글, 펼침/접힘(▲▼)
- [ ] "모두 펼치기" 기능
- [ ] 공정 카드 내부 컨트롤 — 아래 5종 전부 보존:
  - [ ] **칩 단일선택** (예: 부분철거/올철거, 진흥&나노/베코보노/르그랑)
  - [ ] **수량 스테퍼** ±(예: 다운라이트 EA, 시스템에어컨 대수, 방문 조 수, 욕실 개소 수)
  - [ ] **라디오 리스트 선택** (예: 방문 보급형/ABS문/예림, 창호 KCC/LX, 바닥재 목록)

- [ ] 각 공정 카드의 소계 금액 실시간 표시

### 2.3 결과부

- [ ] 시장 평균 대비 경고/안내 메시지 표시

## 4. 광고 슬롯 4종 — 위치 및 규격

광고 슬롯은 4개다. 모두 **본문 흐름(in-flow) 안에** 배치한다. 화면 하단 고정(앵커) 광고는 사용하지 않는다 — 그 자리는 하단 고정 바가 점유한다.

| 슬롯 ID | 규격(모바일) | 방식 | 위치 |
|---|---|---|---|
| **AD-H** | 320×100 | 독점 | 히어로 섹션 ↔ Step 01 사이 |
| **AD-P** | 320×50 | 로테이션 | 공정 카테고리 사이 (동적 — §6 참조) |
| **AD-R** | 300×250 (반응형) | 독점 | 결과부: 공정별 비용 비중 그래프 카드 ↔ 결과 5개 버튼 사이 |
| **AD-F** | 320×50 | 로테이션 | 신뢰 섹션 ↔ 푸터 사이 |

규격은 IAB / 구글 애드센스 표준 규격을 따른다 (광고주가 기존 배너를 그대로 제출할 수 있도록).

### AD-R 위치 주의사항
- AD-R은 "공정별 비용 비중 그래프 카드"의 **바깥**에, 그래프 카드와 결과 버튼 묶음 **사이**에 위치한다.
- 그래프의 "더보기" 펼침/접힘 여부와 무관하게 항상 그래프 카드 *다음, 버튼 묶음 *이전*에 있어야 한다. 더보기를 펼쳐도 광고가 그래프 중간에 끼면 안 된다.

## 5. 슬롯 구조 및 폴백 체인

### 5.1 슬롯 = 컨테이너 원칙
- 각 광고 슬롯은 **컨테이너**다. 그 안의 내용물은 상황에 따라 교체된다.
- 내용물 종류: `자체광고 배너` / `구글 애드센스` / `자사 콘텐츠(house ad)` / `접힘(collapse)`.
- 자체 광고가 팔리면 **코드 수정 없이** 해당 슬롯의 내용물만 교체된다. 애드센스를 하드코딩하지 말 것.

### 5.2 슬롯별 광고주 우선순위 + 폴백 체인

```
AD-H : 자재브랜드 광고 → 턴키업체 광고 → 구글 애드센스 → [자사 신뢰 메시지]
        ※ 어떤 경우에도 비거나 접히지 않는다.

AD-R : 턴키업체 광고 → 구글 애드센스 → [자사 사용팁 + 광고문의 배너]
        ※ 어떤 경우에도 비거나 접히지 않는다.

AD-P : 해당 공정의 단종업체 광고 → [접힘]
        ※ 팔린 광고가 없으면 슬롯이 화면에서 사라진다.

AD-F : 로테이션 광고 → [접힘]
        ※ 팔린 광고가 없으면 슬롯이 화면에서 사라진다.
```

### 5.3 핵심 규칙
- **독점 슬롯(AD-H, AD-R)은 절대 비거나 접히지 않는다.** 채울 광고가 없으면 자사 콘텐츠(house ad)로 채운다. 레이아웃이 미래에 출렁이지 않게 하기 위함.
- **로테이션 슬롯(AD-P, AD-F)은 팔린 광고가 없으면 깔끔하게 접힌다(DOM에서 제거 또는 height 0).**
- 광고주 유형 — AD-H 1순위는 자재 브랜드(KCC·현대L&C·LX 등), AD-R 1순위는 턴키 인테리어 업체, AD-P는 바로 위 공정에 대응하는 단종업체.

### 5.4 현재(런칭 초기) 상태 — 중요
현재는 **자체 광고 0개 + 구글 애드센스 미승인**(애드센스 승인은 일정 트래픽 확보 후 가능) 상태다. 이는 예외가 아니라 **런칭 초기의 정상 상태**로 다룬다.

- **AD-H, AD-R**: 자사·지인 업체 배너로 채운다. (운영자가 직접 준비한 house ad 배너를 슬롯에 넣는 방식. 슬롯 컨테이너 구조상 추후 실제 광고로 교체 가능.)
- **AD-P, AD-F**: 팔린 광고가 없으므로 전부 접힘. 화면에 흔적이 남지 않는다.
- 결과: 광고가 하나도 없어도 사이트는 완성된 모습으로 정상 동작한다. 트래픽이 쌓여 애드센스가 승인되면 그 단계가 자동으로 활성화되고, 자체 광고가 팔리면 그것이 우선 노출된다. **코드 변경 없이 데이터/설정만으로 전환**되어야 한다.

### 5.5 애드센스 정책 주의
- 자체 광고 배너와 애드센스 광고가 같은 화면에 동시 노출될 때, 슬롯 간 충분한 간격을 두고 자체 배너에는 "광고 / Sponsored" 라벨을 명확히 표기한다 (클릭 유도 오인 방지).

## 6. AD-P 동적 배치 로직

AD-P는 화면의 고정 위치가 아니다. **공정 카테고리에 종속된 동적 슬롯**이다.

### 6.1 작동 방식
- 13개 공정 카테고리는 각각 **잠재적으로** 자기 묶음 하단에 AD-P 슬롯을 가질 수 있다.
- 그러나 슬롯은 **그 공정의 단종업체 광고가 실제로 팔렸을 때만** 화면에 나타난다.
- 광고가 없는 공정은 슬롯이 접혀서(collapse) 보이지 않는다.

### 6.2 광고-공정 정합성
- AD-P 슬롯에는 **바로 위 공정 카테고리에 대응하는 업종의 단종업체 광고만** 노출된다.
  - 예: "철거" 카테고리 하단 슬롯 → 철거업체 광고만. "창호" 하단 → 샷시업체만.
- 이로써 13개 단종 직군 전부가 각자 자기 공정 슬롯의 광고주가 될 수 있다 (롱테일 광고주 풀 확보).

### 6.3 노출 캡 (광고판 방지)
- 한 화면(랜딩페이지 1회 로드)에 노출되는 AD-P는 **최대 4~5개로 제한**한다.
- 팔린 AD-P가 캡을 초과하면, 위에서부터 캡 수량만 노출하고 나머지는 노출하지 않는다 (다음 방문 시 로테이션 §8).
- 노출 캡 수치는 하드코딩하지 말고 관리자 설정값으로 둔다.

## 7. 하단 고정 바 동작

하단 고정 바는 **유지한다.** 얼마드나의 정체성인 "실시간 계산기"를 증명하는 핵심 장치다 — 사용자가 공정 토글을 켜고 끌 때 금액이 즉시 바뀌는 것이 항상 보여야 한다. 제거하지 말 것.

### 7.1 높이 압축
- v1의 하단 고정 바는 2줄 구성(라벨 줄 + 큰 금액 줄)으로 세로 픽셀을 다소 많이 차지한다.
- v2에서는 **1줄로 압축**한다: "예상 총 비용" 라벨을 작게 인라인 처리하고 금액을 같은 줄에, 오른쪽에 평당가 + 맨위로(↑) 버튼.
- **금액 숫자의 가독성·크기는 유지**한다 (가장 중요한 정보). 라벨과 평당가만 작게 처리해 높이를 약 30~40% 줄인다.

### 7.2 결과부 도달 시 숨김
- 사용자가 스크롤하여 **결과부(실시간 견적 결과 카드)에 도달하면 하단 고정 바를 숨긴다.**
  - 이유: 결과 카드에 총액이 큰 글씨로 이미 표시되므로 고정 바가 중복된다. 숨기면 결과부 공간도 확보된다.
- 다시 위로 스크롤하여 입력부로 올라가면 고정 바는 **다시 나타난다.**
- 기준선: "실시간 견적 결과 카드"가 뷰포트에 들어오는 시점 (IntersectionObserver 등으로 판정).

## 8. 로테이션 / 노출 방식 (AD-P, AD-F)

- 같은 슬롯 조건(AD-P의 경우 같은 공정)에 광고주가 2곳 이상이면 **로테이션**한다.
- 로테이션은 순수 랜덤이 아니라 **지분(share of voice) 기반**: 광고주가 결제한 금액 비율만큼 노출 비중을 갖는다.
  - 예: 같은 슬롯에 A 50%, B 33%, C 17% 결제 → 노출 비율도 동일.
- **빈도 캡(frequency cap)**: 한 사용자 세션 내에서 같은 광고가 과도하게 연속 노출되지 않도록 한다.
- 노출수(impression)는 광고주별로 카운트한다 — 지분 검증 및 광고주 대시보드(`얼마드나_단종업체광고시스템_개발명세.md` §3.4)의 데이터 소스로 사용.

## 9. 작업 완료 기준 (Definition of Done)

다음을 모두 충족해야 완료다:

1. §2 기능 보존 인벤토리의 **전 항목을 직접 조작하여 검증** — 평형 선택, 등급 변경 시 자재 자동설정, 공정 토글, 칩/스테퍼/라디오/개별토글/직접입력, 예비비, 더보기, 5개 버튼(PDF·엑셀·카톡·내견적저장·이전견적비교), 로그인·저장, 실시간 재계산 — 전부 v1과 동일하게 작동.
2. 광고 슬롯 4개가 §4 위치에 배치되고, §5 폴백 체인대로 동작.
3. 현재 상태(자체광고 0 + 애드센스 미승인)에서 — AD-H·AD-R은 자사 배너로 채워지고, AD-P·AD-F는 접힘. 사이트가 광고 없이도 정상·완결된 모습.
4. AD-P 동적 배치(§6) 및 노출 캡 동작.
5. 하단 고정 바: 1줄 압축 + 스크롤 고정 + 결과부 도달 시 숨김/복귀(§7).
6. 모바일 화면 기준으로 레이아웃 완성, v1 브랜드 톤 유지.
7. 광고 슬롯 추가·디자인 변경 외에 **로직·상태·계산 결과는 v1과 완전히 동일**.

## 10. 절대 원칙 요약

1. **디자인(표현)만 변경. 로직·상태·계산은 불가침.**
2. **v1 기능 100% 보존** — §2 인벤토리가 검증 기준.
3. 광고 슬롯은 컨테이너 구조 — 내용물은 코드 수정 없이 교체 가능.
4. 독점 슬롯(AD-H/AD-R)은 안 비고 안 접힘 — 없으면 자사 콘텐츠. 로테이션 슬롯(AD-P/AD-F)은 없으면 접힘.
5. AD-P는 공정 종속 동적 슬롯, 바로 위 공정에 맞는 단종업체만, 화면당 4~5개 캡.
6. 하단 고정 바는 유지(1줄 압축, 결과부에서 숨김).
7. 모바일 우선.
8. 수치(노출 캡, 요율 등)는 하드코딩 금지 — 설정값.
